# 11 — Spanish ↔ English Glossary

For users playing PoE 2 in Spanish, this glossary maps **localized in-game terms** to their **English equivalents**.

**Critical rule:** all MCP tools (`poe2_db_lookup`, `poe2_wiki_*`, `analyze_character`, etc.) **only accept English terms**. When a Spanish-speaking user asks "¿cómo consigo más crítico?", Claude must:

1. Recognize "crítico" → "critical strike chance".
2. Call the tool with the English term.
3. Translate the result back to Spanish if the user asked in Spanish.

This file is the canonical reference for that translation. **It is not exhaustive** — for unfamiliar terms, use the Spanish PoE 2 Wiki (https://es.poewiki.net/) as a fallback, or query the wiki via `poe2_wiki_search` with the user's best guess at the English term.

## Translation rules for Claude

1. **Input in Spanish → translate in your head → tool call in English.**
2. **Output: respond in the language the user used.** Don't switch languages on the user.
3. **Game item/skill names: keep them in English in the output if that's how poe.ninja / trade site will reference them**, with the Spanish translation in parentheses if helpful.
   - Example: "I'd recommend looking at Mageblood (Sangre Maga). On poe.ninja it's listed as Mageblood."
4. **Be patient with Spanglish.** Spanish players frequently use English game terms directly ("voy a farmear remnants en endgame"). Accept these without correction.
5. **If you're unsure of a translation, ask.** Better to ask "Cuando dices 'forja rúnica', ¿te refieres a Runeforging?" than to guess and break a tool call.

## CRITICAL: `.build` planner files do NOT need ID translation

The new in-game Build Planner (0.5+) reads `.build` JSON files. **In those files:**

- All `id` values (passives like `strength89`, skill gems like `Metadata/Items/Gems/SkillGemEarthquake`, support gems like `Metadata/Items/Gems/SupportGemFastForward`) are **internal references** that the game resolves to the player's localized text automatically. A `.build` with `"id": "strength89"` shows "+5 to Strength" in English clients and "+5 a Fuerza" in Spanish clients **without any change to the file.**
- All `inventory_id` values (`Weapon1`, `BodyArmour1`, etc.) — same thing. The game resolves them locally.
- All `unique_name` values — same thing. Even if the value is the English string "Kalandra's Touch", the in-game tooltip shows the localized name. **Do not translate `unique_name` values.**

**Only these fields get translated** when a user asks "translate this `.build`":
- root `name`
- root `author` (ask first — usually a handle)
- root `description`
- every `additional_text` at any depth

Full rules in `docs/13-build-file-format.md` §8.

This is **the most important consequence of the format design**: one `.build` file works in every language without duplication. The pack benefits from this — when translating, do less, not more.

## What's missing

This glossary covers the most common terms. For:
- **Specific unique item names** in Spanish → query the wiki with the English name; the wiki may have a Spanish variant.
- **Specific skill gem names** → poe2db.tw has Russian (`lang="ru"`) but **no Spanish parameter**. Use English internally.
- **New uniques added day-1 of 0.5** → may not have stable Spanish translations yet; quote English and offer to look up the Spanish.

**If a translation is unverified, label it `[Unverified ES translation]`** and offer to check in-game.
